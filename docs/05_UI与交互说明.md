# UI 与交互说明

## 1. UI 目标

工具 UI 的重点不是炫酷，而是让用户完成三件事：

1. 快速知道当前是否在识别、识别是否稳定。
2. 能标定 ROI、颜色阈值、按钮模板和延迟。
3. 能在出错时看到原因，并一键暂停/急停。

## 2. 主控制台

线框图：

```text
ui/wireframes/main_dashboard_wireframe.png
```

主控制台区域：

| 区域 | 内容 |
|---|---|
| 顶部状态栏 | 当前 Profile、目标窗口、采集 FPS、运行状态 |
| 左侧运行控制 | 选择窗口、选择 ROI、输入模式、提前量、防抖、开始/暂停/标定按钮 |
| 中央预览 | 实时画面和 Debug Overlay |
| 事件流 | 最近 KeyDown/KeyUp/Tap 事件 |
| 右侧指标 | FPS、处理耗时、HSV 阈值、状态机状态、误检计数 |

### 2.1 运行状态

状态建议：

| 状态 | 含义 | UI 表现 |
|---|---|---|
| 待机 | 未采集、未识别 | 开始按钮可用 |
| 标定中 | 用户正在调整 ROI/阈值 | 输入服务强制 dry-run |
| 识别中 | 正常采集和视觉识别 | 显示 FPS、候选框、事件 |
| Dry-run | 只记录事件，不发送按键 | 事件流显示 DRY-RUN 标记 |
| 暂停 | 采集可继续但不触发输入 | 清空所有 Holding 状态 |
| 急停 | 立即释放所有按键并停止输入 | 需要用户手动恢复 |

### 2.2 主按钮

| 按钮 | 行为 |
|---|---|
| 开始识别 | 启动 Capture/Vision/Timing；默认先 dry-run |
| 暂停 | 停止输出输入事件，保留调试显示 |
| 急停 | 立即 KeyUp 所有按住的键，停止输入服务 |
| 进入标定 | 打开标定向导，强制 dry-run |
| 保存 Profile | 保存当前阈值、ROI、模板路径 |

## 3. 标定向导

线框图：

```text
ui/wireframes/calibration_wizard_wireframe.png
```

标定流程共 6 步。

### 3.1 选择窗口

交互：

1. 点击“选择窗口”。
2. 弹出窗口列表。
3. 用户选择目标窗口。
4. 预览区显示窗口画面。

校验：

- 目标窗口不存在时提示重新选择。
- 目标窗口最小化时暂停采集。
- 目标窗口改变尺寸时提示重新标定 ROI。

### 3.2 框选 ROI

交互：

- 用户在预览上拖拽矩形。
- 鼠标滚轮缩放预览。
- 拖拽边缘调整 ROI。
- UI 实时显示坐标 `[x,y,w,h]`。

输出：

```json
"default_prompt_roi": [120, 60, 1040, 600]
```

### 3.3 颜色阈值

交互：

- 点击“采样暗金按钮”。
- 点击“采样亮金按钮”。
- 点击“采样背景光效”。
- 右侧滑块显示 H/S/V 上下限。
- 预览区实时叠加 mask。

用户可以手动调节：

```text
H min / H max
S min / S max
V min / V max
```

显示指标：

```text
mask_area_ratio
candidate_count
false_positive_count
```

### 3.4 模板确认

交互：

- 工具自动从候选按钮中裁剪小图。
- 以网格方式显示候选。
- 用户点击候选并标记为 A/S/W/D/Q/E/SPACE。
- 用户可删除误检候选。

输出文件：

```text
templates/key_button_raw/<key>_<timestamp>.png
templates/letter_gray/<key>_<timestamp>.png
templates/letter_edge/<key>_<timestamp>.png
```

### 3.5 延迟校准

交互：

- UI 显示当前 `input_advance_ms`。
- 用户用 dry-run 模式回放或实时观察。
- 如果触发偏晚，点击“整体提前 +5ms”。
- 如果触发偏早，点击“整体延后 -5ms”。
- 可为短按和长按分别设置提前量。

建议控件：

```text
Tap 提前量：0–60ms
Hold Down 提前量：0–60ms
Hold Up 提前量：0–60ms
```

### 3.6 保存配置

交互：

- 输入 Profile 名称。
- 显示配置摘要。
- 保存 JSON。
- 可导出为 zip，便于迁移。

## 4. Debug Overlay

线框图：

```text
ui/wireframes/debug_overlay_wireframe.png
```

Overlay 显示内容：

| 元素 | 内容 |
|---|---|
| ROI 边框 | 当前检测区域 |
| 按钮圆 | 圆心、半径、key 标签 |
| 外圈 | ring center、ring radius |
| 长条 | bar bbox、tail bbox |
| 分数 | tap_score、hold_start_score、hold_tail_score |
| 状态 | Idle、Holding、Cooldown |
| 事件 | 最近 KeyDown/KeyUp/Tap |
| 性能 | FPS、处理耗时、延迟提前量 |

### 4.1 Overlay 交互

- `F8`：开始/暂停识别。
- `F9`：显示/隐藏 Overlay。
- `F10`：切换 dry-run。
- `F12`：急停并释放所有按键。

热键可在设置中修改。

### 4.2 调试信息层级

建议提供 3 档：

| 档位 | 显示内容 |
|---|---|
| 简洁 | FPS、状态、最近事件 |
| 标准 | 候选框、key、状态、分数 |
| 详细 | mask、圆心、半径、预测时间、模板匹配分 |

## 5. 事件流设计

事件流示例：

```text
16.100  S  KeyDown  score=0.86  reason=hold_start
16.466  W  Tap      score=0.91  reason=tap_alignment
16.900  S  KeyUp    score=0.82  reason=tail_reached
17.000  A  KeyDown  score=0.88  reason=hold_start
```

每条事件点击后可展开：

```text
frame_id
capture_time_ms
vision_time_ms
state_before
state_after
bbox
template_score
predicted_ms_to_hit
```

## 6. 设置页

建议设置项：

| 设置 | 说明 |
|---|---|
| Capture Backend | Windows Graphics Capture / DXGI |
| Capture FPS | 60 / 120 / 自动 |
| 输入模式 | Dry-run / 测试窗口 / 授权目标窗口 |
| HSV 阈值 | H/S/V 上下限 |
| 短按持续时间 | 默认 35ms |
| 输入提前量 | 默认 22ms |
| 连续帧确认 | 默认 2 帧 |
| 防抖时间 | 默认 120ms |
| 日志目录 | CSV/Debug帧输出位置 |

## 7. 错误提示

| 场景 | 提示文案 |
|---|---|
| 未选择窗口 | 请先选择目标窗口 |
| 目标窗口丢失 | 目标窗口不存在，已暂停识别 |
| ROI 无候选 | 当前 ROI 内没有检测到提示，请重新框选 |
| 模板匹配过低 | 候选按钮无法识别为已知按键，请重新采集模板 |
| 长按中目标丢失 | 已安全释放按键，原因：目标丢失 |
| 权限不一致 | 无法向目标窗口发送输入，请以相同权限运行或使用测试模式 |

## 8. 安全交互

- 默认启动为 dry-run。
- 切换到真实输入前，必须确认目标窗口。
- 暂停和急停始终可用。
- 急停会释放所有处于 Holding 的按键。
- 日志记录每次真实输入的原因。
- 标定向导中不允许真实输入。

## 9. 推荐 UI 技术

推荐 WPF：

- 开发效率高。
- 适合桌面工具。
- 绑定配置和状态方便。
- Overlay 可用透明置顶窗口实现。

如果团队熟悉 WinUI 3，也可使用 WinUI 3；但窗口捕获和 Overlay 的工程复杂度会略高。
