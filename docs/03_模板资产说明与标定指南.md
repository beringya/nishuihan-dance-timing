# 模板资产说明与标定指南

## 1. 模板资产来源

本交付包中的模板从用户提供的录屏中抽帧生成，作为初始开发样本。由于录屏只有 30 FPS，并且存在压缩、缩放和强背景光效，模板不应被视为生产环境最终模板。

正式使用时，应在目标电脑和目标窗口中通过标定向导重新采集：

```text
同一分辨率
同一窗口模式
同一画质/亮度/特效设置
同一 UI 缩放
```

## 2. 目录说明

```text
templates/
├─ full_prompt/       # 长按完整提示：顶部圆、竖条、底部字母按钮
templates/key_button_raw/    # 按钮圆/外圈原始模板
templates/letter_gray/       # 灰度字母模板，推荐用于轻量匹配
templates/letter_edge/       # 边缘模板，适合抗亮度变化
templates/letter_binary/     # 二值模板，仅作参考
templates/masks/             # 圆形、长条、文字区域 mask
templates/examples/          # HSV 分割示例
templates/templates_manifest.json
```

## 4. 模板使用建议

### 4.1 按钮候选检测

先用 HSV 和几何方法找到按钮，而不是直接在整张图做模板匹配。

流程：

```text
HSV 金色 mask
→ 轮廓/圆形候选
→ 裁剪按钮中心 48×48
→ 与 letter_gray 或 letter_edge 匹配
→ 得到按键类别
```

这样速度快，也更抗背景干扰。

### 4.2 字母模板匹配

推荐优先级：

1. `letter_gray/`：保留灰度纹理，适合 `cv2.matchTemplate` 的 `TM_CCOEFF_NORMED`。
2. `letter_edge/`：适合强光变化时使用，先做 Canny 边缘。
3. `letter_binary/`：仅作参考，实际可能因压缩和光效不稳定。

匹配范围应限制在按钮中心，不要在全图匹配。

### 4.3 长按模板

`full_prompt/` 中的长按模板适合开发阶段观察结构，不建议生产环境直接整图模板匹配。生产推荐：

```text
底部按钮圆：识别 key 和起点高亮
竖向长条：识别是否为 hold 类型
顶部/尾端圆：估算释放位置
```

也就是说，长按应拆组件检测，而不是把完整长条当作一个大模板。

## 5. 标定流程

### 步骤 1：选择目标窗口

- 用户点击“选择窗口”。
- 工具列出可见窗口。
- 绑定窗口句柄和窗口标题。
- 启动时检查目标窗口是否仍然存在。

### 步骤 2：框选提示区域 ROI

- 在预览画面中拖拽矩形。
- 初始可以使用 `default_prompt_roi`。
- 保存为 `[x, y, w, h]`。
- ROI 应覆盖所有按键提示出现区域，但尽量排除 UI 边角和无关金色元素。

### 步骤 3：颜色阈值标定

建议采样三类区域：

| 样本 | 作用 |
|---|---|
| 暗金按钮 | 估计等待状态 |
| 亮金按钮/外圈 | 估计触发状态 |
| 背景光效 | 估计误检排除 |

标定输出：

```json
"gold_hsv_lower": [13, 50, 70],
"gold_hsv_upper": [45, 255, 255],
"bright_gold_hsv_lower": [18, 80, 145],
"bright_gold_hsv_upper": [42, 255, 255]
```

### 步骤 4：采集按钮模板

对每个 key 至少采集：

```text
等待态按钮 1 张
触发态按钮 1 张
强光背景下按钮 1 张
```

保存为：

```text
templates/key_button_raw/<key>_<state>_<timestamp>.png
```

然后自动生成：

```text
templates/letter_gray/<key>_<state>.png
templates/letter_edge/<key>_<state>.png
```

### 步骤 5：短按时机标定

在 Debug Overlay 中显示：

```text
button_center
ring_center
button_radius
ring_radius
tap_alignment_score
predicted_ms_to_hit
```

调试目标：

```text
tap_alignment_score 在最佳帧附近达到峰值
Tap 事件不早于外圈明显靠近
Tap 事件不晚于评分反馈出现
```

### 步骤 6：长按起点/终点标定

显示两类分数：

```text
hold_start_score：底部按钮/起点高亮程度
hold_tail_score：尾端到达释放区程度
```

调试目标：

```text
KeyDown 在起点高亮刚达标时出现
KeyUp 在尾端到达时出现，而不是等长条消失后
```

### 步骤 7：延迟提前量标定

初始值：

```text
input_advance_ms = 22
```

调参方式：

| 现象 | 调整 |
|---|---|
| 总是偏晚 | 增大提前量，例如 +5ms |
| 总是偏早 | 减小提前量，例如 -5ms |
| 有时重复触发 | 增大 debounce 或 reset 阈值 |
| 长按释放偏晚 | 提高尾端预测权重或降低 tail 阈值 |

## 6. 重新抽取模板脚本

交付包包含：

```text
scripts/extract_templates_from_video.py
```

示例：

```bash
python scripts/extract_templates_from_video.py \
  --video video_1782739957535_m_compressed.mp4 \
  --out ./template_candidates \
  --step-frames 15
```

输出：

```text
candidate_sheet.jpg
cand_000_*.png
cand_001_*.png
...
```

人工筛选后，将好的模板复制到 `templates/` 对应目录，并更新 `templates_manifest.json`。

## 7. 模板质量检查表

| 检查项 | 要求 |
|---|---|
| 按钮圆完整 | 不被大面积遮挡 |
| 字母清晰 | 中心字母可见 |
| 背景少 | 不包含大块非提示金色元素 |
| 尺寸一致 | 统一归一到 64/96 像素 |
| 类型明确 | tap/hold/space/icon 分开存放 |
| 来源记录 | 帧号、时间、坐标写入 manifest |
