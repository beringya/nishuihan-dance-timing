# 开发计划

## 1. 开发目标

交付一款可在 Windows 本地运行的按键时机识别工具。第一版重点是稳定识别、可标定、可调试，不引入大模型。

第一版范围：

- 屏幕/窗口采集。
- HSV 金黄色分割。
- 普通短按和长按候选检测。
- 字母/图标轻量模板匹配。
- 每个按键独立状态机。
- Dry-run 事件日志。
- 授权测试环境下的输入触发。
- 主控制台、标定向导、调试浮层。

不纳入第一版：

- 大模型视觉识别。
- 云端推理。
- 反作弊、风控或权限绕过。
- 无用户确认的后台自动注入。

## 2. 里程碑总览

| 阶段 | 周期 | 目标 | 交付物 |
|---|---:|---|---|
| M0 需求与标定样本 | 2–3 天 | 确认提示类型、键位范围、窗口采集方式 | 标定样本、配置草案 |
| M1 离线视觉原型 | 4–5 天 | 录屏内能检测金色候选、按钮圆、长按条 | Python 原型、debug 帧、事件 CSV |
| M2 状态机与时机判断 | 4–5 天 | 短按/长按 KeyDown/KeyUp 事件稳定 | 状态机库、单元测试、事件回放器 |
| M3 Windows 实时采集 | 5–7 天 | 60/120 FPS 采集，ROI 实时处理 | CaptureService、FrameBuffer、性能报告 |
| M4 UI 与标定向导 | 5–7 天 | 可视化配置 ROI、阈值、模板、延迟 | WPF/WinUI 界面、Profile 管理 |
| M5 输入服务与联调 | 3–5 天 | Dry-run 与测试窗口输入闭环 | InputService、热键急停、事件日志 |
| M6 稳定性优化与打包 | 5–7 天 | 降误检、调性能、打包安装 | 安装包、使用说明、验收报告 |

总周期建议：4–6 周。若只做 Python 离线原型，约 1 周可完成；若要稳定 Windows 小工具，建议至少 4 周。

## 3. 阶段任务拆解

### M0：需求与标定样本

任务：

- 确认目标场景是否为授权环境。
- 列出需要支持的键：A/S/W/D/Q/E/SPACE 等。
- 确认提示是否总是金黄色，是否有主题/亮度变化。
- 确认用户屏幕分辨率、缩放、窗口模式。
- 从录屏或现场画面采集样本模板。

验收：

- 至少有 5 类按钮模板、3 类长按模板、1 套 HSV 初始阈值。
- 确认短按和长按的视觉触发规则。

### M1：离线视觉原型

任务：

- 读取录屏帧。
- 裁剪 ROI。
- HSV 分割金色区域。
- 提取轮廓、圆形、长条。
- 保存候选框图和 contact sheet。
- 初步区分 tap / hold 候选。

验收：

- 对已知片段能稳定检测到按钮圆和长按条。
- Debug 图能展示候选框和分数。
- 可输出 `prototype_events.csv`。

### M2：状态机与时机判断

任务：

- 实现 `Idle / Holding / Cooldown` 状态机。
- 短按使用外圈重合分数触发。
- 长按使用起点分数 KeyDown、尾端分数 KeyUp。
- 增加连续帧确认、防抖、reset 滞回。
- 加入 `input_advance_ms` 提前量。

验收：

- 同一提示不重复触发。
- 长按不会提前松开，也不会等完全消失才松。
- 可通过参数调早/调晚。

### M3：Windows 实时采集

任务：

- 实现 Windows Graphics Capture。
- 备选实现 DXGI Desktop Duplication。
- FrameBuffer 使用无锁或轻锁队列。
- 采集帧带高精度时间戳。
- 支持目标窗口选择和 ROI 坐标转换。

验收：

- 60 FPS 稳定运行。
- 120 FPS 模式下 ROI 视觉处理不阻塞采集。
- 总视觉处理耗时目标 < 4ms。

### M4：UI 与标定向导

任务：

- 主控制台：启停、窗口选择、配置选择、实时指标。
- ROI 标定：拖拽矩形、缩放预览、保存坐标。
- 颜色标定：取亮/暗样本，预览 HSV mask。
- 模板标定：采集按钮样本，确认 key 标签。
- 延迟标定：记录早/晚反馈，调整提前量。
- Debug Overlay：候选框、分数、状态、事件流。

验收：

- 非开发人员可以完成基本标定。
- 所有阈值修改可实时预览。
- 配置可保存和恢复。

### M5：输入服务与联调

任务：

- 输入服务先实现 dry-run。
- 测试窗口中实现真实输入验证。
- 增加目标窗口校验。
- 增加全局热键：开始/暂停/急停。
- 事件日志记录输入时间、视觉时间、延迟。

验收：

- Dry-run 事件与视觉状态一致。
- 测试窗口输入顺序正确。
- 急停立即释放所有按住的键。

### M6：稳定性优化与打包

任务：

- 回放多段录屏，整理误检案例。
- 优化 HSV 阈值和几何规则。
- 优化 CPU 占用和内存占用。
- 输出用户说明、故障排查、默认配置。
- 打包为安装版或免安装版。

验收：

- 标定后误触发率低于可接受阈值。
- 启动、暂停、退出、急停无卡死。
- 日志足以回溯每次按下/弹起原因。

## 4. 人员分工建议

| 角色 | 工作内容 |
|---|---|
| Windows 客户端工程师 | 采集、输入、WPF/WinUI、打包 |
| 视觉算法工程师 | OpenCV 检测、模板匹配、阈值调优 |
| 测试工程师 | 标定样本、回放测试、误检统计 |
| 产品/设计 | UI 流程、标定交互、状态提示 |

小团队可由 1 名 Windows 工程师 + 1 名视觉工程师完成。

## 5. 验收用例

### 用例 1：普通短按

步骤：

1. 播放包含 W/A/S/D 短按的录屏。
2. 开启 dry-run。
3. 检查事件 CSV。

期望：

- 每个短按只产生一次 Tap。
- Tap 时间在外圈重合前后 20ms 内。
- 不因评分文字或背景光效重复触发。

### 用例 2：长按

步骤：

1. 播放包含 A/Q/S/W/D 长按的录屏。
2. 检查 KeyDown 和 KeyUp。

期望：

- 起点高亮时 KeyDown。
- 尾端到达时 KeyUp。
- 长按期间没有重复 KeyDown。
- 目标丢失时可以安全 KeyUp。

### 用例 3：多键并发

步骤：

1. 使用 15.5s–17s 片段。
2. 多个按键同时出现。

期望：

- 每个 key 独立维护状态。
- S/W/A 等不会互相阻塞。
- 长按和短按可以交错处理。

### 用例 4：暂停与急停

步骤：

1. 长按保持中按下急停热键。

期望：

- 立即释放所有已按下按键。
- 状态机进入安全待机。
- 日志记录急停原因。

## 6. 风险排期

| 风险 | 影响 | 预留时间 |
|---|---|---:|
| 分辨率/缩放导致模板失配 | 识别不稳定 | 2 天 |
| 背景光效与金色提示重叠 | 误检 | 3 天 |
| 长按尾端不明显 | 弹起偏晚/偏早 | 2–3 天 |
| 实时采集帧率不稳 | 时机漂移 | 2 天 |
| 用户环境差异 | 配置失效 | 2 天 |

## 7. 版本路线

### V0.1 离线验证版

- 读取录屏。
- 输出候选框和事件日志。
- 不发送按键。

### V0.2 实时调试版

- 实时采集。
- UI 显示分数和候选框。
- Dry-run 事件流。

### V0.3 授权测试输入版

- 测试窗口输入。
- 支持短按、长按、急停。
- 保存 profile。

### V1.0 稳定工具版

- 完整标定向导。
- 多 profile。
- 性能优化。
- 使用文档和打包发布。
