# 视频时机观察摘要

## 1. 视频属性

- 分辨率：1280×694
- 帧率：30 FPS
- 时长：约 107.7s
- 离线抽帧时机精度：约 ±33ms

## 2. 关键观察

### 普通短按

普通短按的最佳点出现在外圈/亮点与字母按钮重合时。评分文字或爆光反馈是命中后的结果，不能作为触发信号。

开头 W 短按样例：

```text
约 0.30s–0.33s 外圈和按钮明显重合，可视为最佳短按窗口。
约 0.40s 后已进入反馈/消散阶段，偏晚。
```

### 长按

长按应分为按下和弹起两个独立时机：

```text
按下：起点字母圆高亮或头部到达判定位置。
弹起：尾端/长条结束点到达释放位置。
```

开头 A 长按样例：

```text
约 1.10s–1.17s：A 起点高亮，适合 KeyDown。
约 2.33s–2.37s：尾端到达/释放窗口，适合 KeyUp。
约 2.40s 以后：已接近反馈/消失，作为弹起依据会偏晚。
```

### 多键片段

15.5s–17.0s 片段中，短按、长按和多键并发同时出现。每个 key 必须使用独立状态机，不能用全局状态阻塞其他按键。

## 3. 推荐检测信号

| 类型 | 首选信号 | 备选信号 |
|---|---|---|
| 短按 | 外圈半径与按钮半径接近、圆心接近 | 按钮 ROI 高亮比例 |
| 长按按下 | 起点按钮 bright_gold 比例过阈值 | 头部/起点到达判定区域 |
| 长按弹起 | 尾端圆/长条末端到达释放区域 | 长条消失前的长度/速度预测 |
| key识别 | 按钮中心灰度/边缘模板匹配 | 人工标定固定通道 |

## 4. 技术路线结论

不需要本地大模型。推荐：

```text
Windows实时采集
→ ROI裁剪
→ HSV金色分割
→ 圆形/长条/尾端检测
→ 小模板识别字母
→ 状态机判断按下/弹起
→ dry-run日志或授权输入
```
